Журнал внесенных изменений в MLdr7188.exe (full duplex):

Build 046 (2006-02-14):
* Изменения в протоколе обмена: изменен формат пакета данных, заменена функция подсчета 
  контрольной суммы с ModBusCRC на PPP_FCS16 (RFC1662), в связи с чем для работы с этой
  версией необходимо обновление прошивки контроллера. 

Build 043 (2006-01-28):
* Обнаружена временами нестабильность работы MLdr7188 (блокировка COM-порта) на 
  скоростях передачи данных 57600 бит/сек и более. Пока не устранено.
  Рекомендуется ограничение скорости порта не более 38400 бит/с.

Build 042 (2006-01-25):
+ Добавлен статический параметр AnswerTimеout - максимальное время (в секундах) отсутствия ответа
  контроллера, по прошествии которого связь автоматически разрывается. Полезно в случае "подвисающей"
  связи или некорректной работы контроллера для экономии финансовых средств при повременной оплате
  канала связи. Значение по умолчанию = 8.
+ Для нормальной работы CDMA2000-терминала (модема) "AnyDATA DTG-450" в процедуру принудительного 
  разрыва связи добавлена переинициализация модема (иначе не происходил сброс сигнала обнаружения 
  несущей).

Build 041 (2005-09-27):
+ После получении новых событий с КП каждые 5 сек окно программы перемещается на передний план,
  соответствующая вкладка свойств КП активизируется до тех пор, пока оператором не будет нажата
  кнопка "Квитировать"
* Исправлено: параметр RedialInterval считывается из файла статических параметров (считывался из файла
  динамических параметров)

Build 040 (2005-09-26):
+ Доработан модуль приемопередатчика (теперь для каждого контроллера заводится отдельная запись 
  с параметрами и очередями пакетов ARQ/SREJ-протокола). Теперь программа должна работать корректно
  и при опросе нескольких контроллеров.

Build 039 (2005-09-22):
* Устранена причина проблем с форматом даты/времени при сохранении в файле параметров, а также,
  предположительно, устранено его (формата) произвольное изменение при работе программы из-за считывания
  региональных настроек Windows по событию WM_INICHANGE.
* Исправлено: частый передозвон при таймауте ответа контроллера. Теперь после автоматического разрыва
  связи следующая попытка дозвона будет не ранее, чем через RedialInterval секунд (указывается в файле 
  статических параметров).
